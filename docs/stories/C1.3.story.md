## Context

O app é uma PWA instalável (`@serwist` configurado, `PwaInstallBanner` existe) mas não envia push notifications. O service worker atual (`src/sw.ts`) faz apenas precaching — não tem handler de eventos `push`.

Push notifications são pré-requisito para C1.4 (alertas de orçamento). Sem elas, o usuário só descobre que ultrapassou um orçamento quando ele mesmo abre o app — o que elimina o valor principal da feature de budgets.

Todos os concorrentes sérios (Mobills, Organizze, YNAB, Monarch Money) têm alertas de orçamento por push. É a "última milha" da feature de orçamento que já existe no app.

Esta story entrega apenas a infraestrutura: tabela de subscriptions, service worker com push handler, e as duas API routes necessárias. A lógica de _quando_ enviar push fica em C1.4.

---

## User Story

> Como usuário,
> quero autorizar o app a me enviar notificações push,
> para receber alertas financeiros importantes
> sem precisar abrir o app manualmente.

### AC1: Migration — tabela `push_subscriptions`

- [ ] Nova migration `016_push_subscriptions.sql` em `supabase/migrations/`
- [ ] Schema:
  ```sql
  CREATE TABLE push_subscriptions (
    id         uuid DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id    uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    endpoint   text NOT NULL,
    keys       jsonb NOT NULL,  -- { p256dh, auth }
    created_at timestamptz DEFAULT now(),

  );
  ```
- [ ] RLS: `user_id = auth.uid()` para SELECT, INSERT, DELETE
- [ ] Índice em `user_id` para buscar subscriptions por usuário no envio
- [ ] **Verificação:** `supabase db push` sem erros

### AC3: Dependência `web-push` instalada

- [ ] `web-push` instalado como dependency: `npm install web-push`
- [ ] `@types/web-push` instalado como devDependency
- [ ] **Verificação:** `import webpush from 'web-push'` em uma API route sem erro de TypeScript

### AC4: Service worker — handler de push events

- [ ] `src/sw.ts` adiciona listener para o evento `push`:
  ```typescript
  self.addEventListener('push', (event) => {
    const data = event.data?.json() ?? {};
    event.waitUntil(
      self.registration.showNotification(data.title ?? 'Finansim', {
        body: data.body ?? '',

        data: { url: data.url ?? '/dashboard' },
      })

  ```
- [ ] Listener para `notificationclick`: abre (ou foca) a URL contida em `notification.data.url`
- [ ] **Verificação:** DevTools → Application → Service Worker → Push → notificação aparece com título e corpo

### AC5: API route — POST /api/push/subscribe

- [ ] `src/app/api/push/subscribe/route.ts`
- [ ] Autenticação obrigatória (401 se não logado)
- [ ] Body: `{ endpoint: string, keys: { p256dh: string, auth: string } }`
- [ ] Upsert na tabela `push_subscriptions` (ON CONFLICT DO UPDATE SET keys = EXCLUDED.keys)
- [ ] Retorna 200 com `{ subscribed: true }`
- [ ] **Verificação:** POST com subscription válida → linha criada/atualizada no banco

### AC6: API route — POST /api/push/send (internal)

- [ ] `src/app/api/push/send/route.ts`
- [ ] **Rota interna** — verifica `Authorization: Bearer ${SUPABASE_SERVICE_ROLE_KEY}` no header
  - Retorna 401 se token ausente ou incorreto
  - Nunca acessível pelo frontend diretamente
- [ ] Body: `{ user_id: string, title: string, body: string, url?: string }`
- [ ] Busca todas as subscriptions do `user_id` na tabela `push_subscriptions`
- [ ] Para cada subscription: chama `webpush.sendNotification()` com VAPID configurado
- [ ] Se `webpush.sendNotification()` retornar 410 (Gone) ou 404: deletar a subscription do banco (endpoint expirado)
- [ ] Retorna `{ sent: number, failed: number }`
- [ ] **Verificação:** POST com service role key + user_id com subscription → notificação recebida no browser

### AC7: Solicitar permissão de push ao usuário

- [ ] Exibir prompt de permissão após **onboarding completo** (quando `OnboardingBanner` seria dispensado) OU na primeira visita à página `/more/orcamentos`
- [ ] Usar componente `PushPermissionBanner` (novo) — banner não-intrusivo com:
  - Texto: "Ative notificações para receber alertas de orçamento"
  - Botão "Ativar" e botão "Agora não"
  - Se clicado "Agora não": salvar flag no `localStorage` e não exibir por 7 dias
  - Se permissão negada pelo browser: esconder permanentemente (sem spam)
- [ ] Ao conceder permissão: chamar `pushManager.subscribe()` com `NEXT_PUBLIC_VAPID_PUBLIC_KEY` e então `POST /api/push/subscribe`
- [ ] **Verificação:** Completar onboarding → banner aparece; clicar "Ativar" → permissão solicitada pelo browser; conceder → subscription salva no banco

## Tech Notes for @dev

### Gerar chaves VAPID (executar uma vez, salvar no .env)

```bash
npx web-push generate-vapid-keys
# Saída:
# Public Key: BNxxxxxxx...
# Private Key: xxxxxxx...
```

Salvar no `.env.local` e nas variáveis de ambiente do Vercel (Settings → Environment Variables).

### Configurar web-push no servidor

```typescript
// src/lib/push/config.ts
import webpush from 'web-push';

webpush.setVapidDetails(
  process.env.VAPID_SUBJECT!,
  process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!,
  process.env.VAPID_PRIVATE_KEY!,
);

export { webpush };
```

### Subscribe no frontend

```typescript
// src/lib/push/subscribe.ts
export async function subscribeToPush(): Promise<boolean> {
  if (!('serviceWorker' in navigator) || !('PushManager' in window)) return false;

  const registration = await navigator.serviceWorker.ready;
  const subscription = await registration.pushManager.subscribe({
    userVisibleOnly: true,
    applicationServerKey: process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY,
  });

  const { endpoint, keys } = subscription.toJSON() as {
    endpoint: string;
    keys: { p256dh: string; auth: string };
  };

  const res = await fetch('/api/push/subscribe', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ endpoint, keys }),
  });

  return res.ok;
}
```

### Handler de push no service worker (src/sw.ts)

O `src/sw.ts` usa Serwist — adicionar os listeners antes de `serwist.addEventListeners()`:

```typescript
self.addEventListener('push', (event: PushEvent) => {
  const data = event.data?.json() ?? {};
  event.waitUntil(
    self.registration.showNotification(data.title ?? 'Finansim', {
      body: data.body ?? '',
      icon: '/icons/icon-192x192.png',
      data: { url: data.url ?? '/dashboard' },
    })
  );
});

self.addEventListener('notificationclick', (event: NotificationEvent) => {
  event.notification.close();
  event.waitUntil(
    clients.matchAll({ type: 'window' }).then((clientList) => {
      const url = event.notification.data?.url ?? '/dashboard';
      const existing = clientList.find((c) => c.url === url && 'focus' in c);
      if (existing) return existing.focus();
      return clients.openWindow(url);
    })
  );
});
```

### API /api/push/send — validação interna

```typescript
const authHeader = request.headers.get('Authorization');
const expectedToken = `Bearer ${process.env.SUPABASE_SERVICE_ROLE_KEY}`;
if (authHeader !== expectedToken) {
  return Response.json({ error: 'Não autorizado' }, { status: 401 });
}
```

Esta rota é chamada exclusivamente pela Edge Function `check-budget-alerts` (C1.4) — nunca exposta ao frontend.

### Graceful degradation (VAPID ausente)

```typescript
// Em /api/push/subscribe:
if (!process.env.VAPID_PRIVATE_KEY) {
  return Response.json({ subscribed: false, reason: 'push_disabled' }, { status: 200 });
}
```
