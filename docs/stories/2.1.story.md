## Context

Com o Epic 1 completo, transacoes importadas ficam com `category='outros'` e `category_source='pending'`. Esta story implementa o motor de categorizacao automatica com IA — o diferencial tecnico central do produto.

A arquitetura e de dois passos (arch section 4.2):
- **Fase sync** (`/api/import`) ja existe (Story 1.3) e retorna `status: 'categorizing'`
- **Fase async** (Supabase Edge Function `categorize-import`) e o que esta story implementa

A categorizacao usa uma hierarquia de 3 camadas para minimizar custo de IA (arch section 5.1):
1. **Dicionario do usuario** (`category_dictionary`) — regras aprendidas com correcoes (Story 2.2 popula)
2. **Cache global** (`category_cache`) — reutiliza categorizacoes cross-user
3. **Claude Haiku** — batch para o que nao esta em cache

A UI do importador ja exibe `status: 'categorizing'`. Esta story faz a categorizacao real acontecer e notifica o cliente via Supabase Realtime quando concluida.

**PRD refs:** FR6 (100% automatico), FR7 (contexto brasileiro), FR8 (aprender com correcoes), FR9 (dicionario), NFR1 (< 30s para 500 transacoes)
**Arch refs:** Import Pipeline (section 4.2), AI Categorization (section 5.1), Prompt Design (section 5.2), Cost Control (section 5.5)

## User Story

> Como usuario,
> quero que minhas transacoes sejam categorizadas automaticamente apos importar o extrato,
> para que eu saiba para onde vai meu dinheiro sem precisar classificar nada manualmente.

## Tech Notes for @dev

### Edge Function vs API Route
- **Use Edge Function** para categorizacao: processo pode levar 10-30s (Vercel Hobby tem timeout de 10s)
- **API Route** somente para parsing/save (< 5s) + invocar a Edge Function

### Invocando Edge Function a partir da API Route
```typescript
// src/app/api/import/route.ts — apos salvar transacoes:
const supabaseAdmin = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
)
// Fire-and-forget: nao bloqueia a resposta ao cliente
supabaseAdmin.functions.invoke('categorize-import', {
  body: { importId, userId }
}).catch(err => console.error('Edge Function invoke failed:', err))
```

### Normalizacao de descricao
```typescript
// Usar a funcao SQL do banco (evita duplicar logica):
const { data } = await supabase.rpc('normalize_description', { raw: description })
// Ou normalizar em JS para batch:
function normalizeDescription(raw: string): string {
  return raw
    .toLowerCase()
    .normalize('NFD').replace(/[\u0300-\u036f]/g, '') // remove acentos
    .replace(/[^a-z0-9 ]/g, '') // remove especiais
    .replace(/\s+/g, ' ')       // colapsa espacos
    .trim()
}
```

### Batch Claude Haiku — custo estimado
- ~500 transacoes por chamada → ~2.000 tokens input + ~500 tokens output
- Custo: ~$0.01-0.02 por extrato (Haiku: $0.25/MTok input, $1.25/MTok output)
- NUNCA chamar Haiku por transacao individual

### Modelo correto
- **Categorizacao batch:** `claude-haiku-4-5-20251001` (CLAUDE.md: "Haiku para batch")
- **Chat:** `claude-sonnet-4-6` (Story 3.1)

### Supabase Realtime no cliente
```typescript
// src/components/import/import-progress.tsx
const channel = supabase
  .channel(`import-${importId}`)
  .on('postgres_changes', {
    event: 'UPDATE',
    schema: 'public',
    table: 'imports',
    filter: `id=eq.${importId}`
  }, (payload) => {
    if (payload.new.status === 'completed') { /* ... */ }
  })
  .subscribe()

return () => { supabase.removeChannel(channel) }
```

### Tratamento de erros na Edge Function
- Em caso de falha da Claude API: marcar import como `status='failed'` com `error_message`
- Nao silenciar erros — o usuario precisa saber que deve tentar novamente
- Log de erros: `console.error()` — aparece no Supabase Edge Function logs
