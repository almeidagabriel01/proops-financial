## Context

A tabela `transactions` não possui campo `notes`. Usuários não têm onde registrar contexto adicional sobre uma transação — por exemplo, "reembolso da empresa", "presente de aniversário da Maria", ou "parcela do conserto do carro".

Todos os concorrentes sérios (Copilot Money, Monarch Money, YNAB, Mobills) oferecem notas em transações. É uma das features mais simples de implementar e elimina uma reclamação direta: o usuário precisa lembrar o contexto de uma transação semanas depois.

A PATCH `PATCH /api/transactions/[id]` já existe e tem padrão extensível — basta adicionar `notes` ao `UpdateTransactionBody` e ao `updates` object. O `TransactionDetail` sheet já existe em `src/components/transactions/transaction-detail.tsx`.

---

## User Story

> Como usuário,
> quero adicionar uma nota livre em qualquer transação,
> para registrar contexto adicional (reembolso, presente, motivo específico)
> e encontrá-lo quando consultar meu histórico financeiro.

## Tech Notes for @dev

### Migration

```sql
-- supabase/migrations/014_transaction_notes.sql
ALTER TABLE transactions ADD COLUMN notes text;
```

Sem NOT NULL, sem default — retrocompatível com todas as transações existentes.

### Extensão da PATCH route

```typescript
// src/app/api/transactions/[id]/route.ts
interface UpdateTransactionBody {
  date?: string;
  description?: string;
  amount?: number;
  type?: 'credit' | 'debit';
  category?: string;
  notes?: string | null;  // NOVO
}

// No bloco de validações:
if (body.notes !== undefined) {
  if (body.notes !== null && body.notes.length > 500) {
    return Response.json({ error: 'Nota muito longa (máximo 500 caracteres)' }, { status: 400 });
  }
  updates.notes = body.notes; // null limpa, string salva
}
```

### TransactionDetail — edição inline com onBlur

```typescript
// Salvar apenas quando o valor mudou
const [noteValue, setNoteValue] = useState(transaction.notes ?? '');
const [savedNote, setSavedNote] = useState(transaction.notes ?? '');

async function handleNoteBlur() {
  if (noteValue === savedNote) return; // sem mudança

```

#### MINORs

**mn1 — AC4 ambíguo: ícone vs. texto truncado na lista**
AC4 menciona "exibe ícone de nota" E "truncada em 60 caracteres" como itens separados. Não está claro se o design final é: (a) apenas ícone na lista + texto completo no detalhe, ou (b) ícone + texto truncado visível na lista. @dev deve escolher e documentar — recomendado (a) para manter a `TransactionItem` compacta em mobile.

**mn2 — Sem AC para estado de erro do onBlur save**
AC3 especifica loading + checkmark de sucesso, mas não feedback de falha. Se o PATCH falhar (timeout, rede), o usuário não tem indicação visual. Adicionar: ao falhar, restaurar o valor original e exibir toast de erro breve ("Erro ao salvar nota — tente novamente").
