## Context

PocketGuard (US) deve 4.8 estrelas no App Store a uma única feature: "In My Pocket" — um número simples que responde "quanto posso gastar hoje de forma segura?". É o item mais elogiado em reviews de PFM internacionalmente: "um número simples que me diz o que preciso saber".

O Finansim tem todos os dados necessários para calcular isso: receitas e despesas do mês (já exibidas nos `SummaryCards`) e as contas agendadas para o restante do mês (tabela `scheduled_transactions` com `status = 'pending'`).

A fórmula é simples:
```
saldo_mes = receitas_mes - despesas_mes
contas_pendentes = soma scheduled_transactions (pending, due_date até fim do mês)
safe_to_spend = saldo_mes - contas_pendentes
safe_to_spend_diario = safe_to_spend / dias_restantes_no_mes
```

O card fica abaixo dos `SummaryCards` existentes no dashboard — é a resposta direta à pergunta mais comum dos usuários. Disponível para Basic e Pro.

---

## User Story

> Como usuário,
> quero ver no dashboard quanto posso gastar hoje de forma segura,
> considerando meu saldo do mês e as contas que ainda tenho a pagar,
> para tomar decisões de consumo sem precisar fazer cálculos manualmente.

## Tech Notes for @dev

### Função de cálculo

```typescript
// src/lib/dashboard/safe-to-spend.ts

export interface SafeToSpendInput {
  totalIncome: number;
  totalExpenses: number;
  pendingBills: Array<{ amount: number; due_date: string }>;
  referenceDate?: Date;
}

export interface SafeToSpendResult {
  safeToSpend: number;
  safeToSpendDaily: number;
  balanceMonth: number;
  pendingTotal: number;
  daysRemaining: number;
  pendingBillsCount: number;
}

export function calculateSafeToSpend(input: SafeToSpendInput): SafeToSpendResult {
  const ref = input.referenceDate ?? new Date();
  const endOfMonth = new Date(ref.getFullYear(), ref.getMonth() + 1, 0); // último dia do mês
  const daysRemaining = Math.max(
    1,
    Math.ceil((endOfMonth.getTime() - ref.getTime()) / (1000 * 60 * 60 * 24)) + 1
  );

  // Filtrar apenas contas com due_date até o fim do mês atual
  const currentYearMonth = `${ref.getFullYear()}-${String(ref.getMonth() + 1).padStart(2, '0')}`;
  const relevantBills = input.pendingBills.filter(b =>
    b.due_date.startsWith(currentYearMonth) || b.due_date <= endOfMonth.toISOString().slice(0, 10)
  );

  const pendingTotal = relevantBills.reduce((sum, b) => sum + b.amount, 0);
  const balanceMonth = input.totalIncome - input.totalExpenses;
  const safeToSpend = balanceMonth - pendingTotal;
  const safeToSpendDaily = safeToSpend / daysRemaining;

  return {
    safeToSpend,
    safeToSpendDaily,
    balanceMonth,
    pendingTotal,
    daysRemaining,
    pendingBillsCount: relevantBills.length,
  };
}
```

## Out of Scope

- Incluir saldo de investimentos no cálculo
- "Safe-to-spend" para meses futuros
- Configuração do cálculo pelo usuário (ex: reservar % para poupança)
- Comparativo de safe-to-spend mês a mês
- Notificação push quando safe-to-spend cai abaixo de um threshold
- Incluir parcelas futuras no cálculo (apenas scheduled_transactions por ora)
