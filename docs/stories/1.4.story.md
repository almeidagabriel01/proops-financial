## Context

Com transacoes importadas (Story 1.3), o usuario precisa visualizar seus dados. Esta story entrega a primeira experiencia real do produto: uma lista de transacoes navegavel e um dashboard basico com resumo financeiro.

Neste ponto do MVP, a categorizacao ainda e stub (`category='outros'`). O dashboard deve ser visualmente impactante MESMO SEM categorizacao — agrupando por dia/semana, mostrando receitas vs despesas, saldo. A versao com graficos de categorias vem no Epic 2.

**Decisao de produto confirmada:** O dashboard gratuito deve ser visualmente completo e bonito, nao um placeholder degradado. Este e o primeiro contato real do usuario com o valor do app.

**PRD refs:** FR10 (dashboard consolidado), FR11 (totais), FR12 (visualmente impactante no free), FR13 (lista com filtros)
**Arch refs:** Frontend Architecture (section 7), Component Strategy (section 7.1), Data Fetching (section 7.2), Mobile Layout (section 7.3)

## User Story

> Como usuario,
> quero ver todas as minhas transacoes importadas organizadas e um resumo do meu mes,
> para que eu tenha visibilidade de para onde vai meu dinheiro.

## Acceptance Criteria

### AC1: Lista de transacoes com scroll infinito
- [ ] `src/app/(app)/transactions/page.tsx` — exibe lista de transacoes
- [ ] Cada item mostra: data, descricao, valor (colorido: verde para credito, vermelho para debito), tipo (entrada/saida)
- [ ] Scroll infinito (carregar mais ao chegar no fim — 50 transacoes por pagina)
- [ ] Ordenacao por data decrescente (mais recente primeiro)
- [ ] Performance: primeira carga < 1s com 500 transacoes no banco
- [ ] **Verificacao:** Importar extrato com 200+ transacoes; lista carrega rapido; scroll infinito funciona

### AC2: Filtros funcionais
- [ ] Filtro por periodo: mes atual (default), mes anterior, customizado (date picker)
- [ ] Filtro por tipo: todos (default), receitas, despesas
- [ ] Filtros aplicados via query params (persistem em refresh)
- [ ] UI: filtros em barra fixa acima da lista, design compacto para mobile
- [ ] Combinacao de filtros funciona (ex: despesas do mes anterior)
- [ ] **Verificacao:** Aplicar cada filtro individualmente e combinados; resultados corretos

### AC3: Busca por texto
- [ ] Campo de busca na descricao das transacoes
- [ ] Busca case-insensitive, sem acentos (usar `ilike` no Supabase ou filtro client-side)
- [ ] Debounce de 300ms para evitar queries excessivas
- [ ] **Verificacao:** Buscar "uber" retorna todas as transacoes com "uber" na descricao

### AC4: Dashboard basico — Cards de resumo
- [ ] `src/app/(app)/dashboard/page.tsx` — Server Component

- [ ] Periodo default: mes atual
- [ ] `src/components/dashboard/summary-cards.tsx` — componente reutilizavel
- [ ] **Verificacao:** Importar extrato; dashboard mostra totais corretos; formatacao BRL correta

## Tech Notes for @dev

### Data Fetching Strategy
- **Dashboard (`/dashboard`):** Server Component. Fetch dados no servidor com Supabase server client. Rapido, SEO-friendly (nao relevante aqui, mas e o padrao correto).
- **Transaction List (`/transactions`):** Pode ser Server Component com filtros via search params, ou Client Component com hooks. Recomendacao: Client Component para interatividade de filtros + scroll infinito.

### Queries Supabase (exemplos)
```typescript
// Summary para dashboard
const { data } = await supabase
  .from('transactions')
  .select('amount, type')
  .gte('date', startOfMonth)
  .lte('date', endOfMonth)

const income = data.filter(t => t.type === 'credit').reduce((sum, t) => sum + t.amount, 0)
const expenses = data.filter(t => t.type === 'debit').reduce((sum, t) => sum + Math.abs(t.amount), 0)

// Lista com paginacao
const { data, count } = await supabase
  .from('transactions')
  .select('*', { count: 'exact' })
  .order('date', { ascending: false })
  .range(offset, offset + limit - 1)
```

### Scroll Infinito
- Usar `IntersectionObserver` para detectar quando o usuario chega ao fim da lista
- Carregar proxima pagina (50 itens) e append na lista
- Mostrar spinner enquanto carrega
- Parar quando `count` total for atingido

### Graficos
- **Recharts** e a melhor opcao: leve, React-native, responsivo
- Para MVP: um unico grafico de barras (receita vs despesa por semana)
- Nao usar Chart.js (mais pesado, menos integracao com React)

### Formatacao BRL
```typescript
// src/lib/utils/format.ts
export function formatCurrency(amount: number): string {
  return new Intl.NumberFormat('pt-BR', {
    style: 'currency',
    currency: 'BRL'
  }).format(amount)
}

export function formatDate(date: string): string {
  return new Intl.DateTimeFormat('pt-BR', {
    day: '2-digit',
    month: 'short',
    year: 'numeric'
  }).format(new Date(date))
}
```

### Mobile Layout
- Bottom nav: `fixed bottom-0` com `z-50`, `h-16`
- Page content: `pb-16` para compensar bottom nav
- Cards de resumo: `grid grid-cols-1 sm:grid-cols-3` — empilhados em mobile, lado a lado em desktop
- Lista: `divide-y` com items padded para touch

### shadcn/ui Components (instalar)
- `Card` — summary cards
- `Input` — campo de busca
- `Select` — filtro de periodo/tipo
- `Button` — CTAs
- `Skeleton` — loading states
- `Badge` — tipo de transacao

### Empty State

- Ex: `<FileX2 />` ou `<Inbox />` com texto e botao
