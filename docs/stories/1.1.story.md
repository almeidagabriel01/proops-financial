## Context

Esta e a primeira story do projeto. Nada existe ainda — nenhum codigo, nenhum repositorio, nenhum banco de dados. O objetivo e criar toda a fundacao tecnica: projeto Next.js com App Router, conexao com Supabase (PostgreSQL + Auth + Storage), configuracao de ambiente, tooling de qualidade de codigo, e um health check funcional.

Tudo que vier depois (auth, import, dashboard, IA) depende desta story estar 100% correta. Erros aqui se propagam para todo o projeto.

**PRD refs:** FR20 (Supabase Auth setup), NFR8-10 (infra), NFR12 (custo), CON1-2 (dev solo, budget)
**Arch refs:** Project Structure (section 2), Database Schema (section 3), Env Vars (section 9.2), Deployment (section 10)

## Tech Notes for @dev

### Supabase Setup
1. Criar projeto em `app.supabase.com` (free tier)
2. Instalar Supabase CLI: `npm install -g supabase`
3. `supabase init` na raiz do projeto
4. Copiar TODAS as 8 tabelas + indexes + constraints + functions + triggers + RLS do `docs/architecture/architecture.md` section 3 para `supabase/migrations/001_initial_schema.sql`
5. `supabase db push` para aplicar

### Supabase Clients
- **Browser client** (`client.ts`): `createBrowserClient()` do `@supabase/ssr` — funciona com cookies do Next.js, respeita RLS
- **Server client** (`server.ts`): `createServerClient()` com service role key — bypassa RLS, so usar em API routes
- Instalar: `@supabase/supabase-js` + `@supabase/ssr`

### shadcn/ui
- Seguir wizard: `npx shadcn@latest init`
- Paleta: tons azul escuro/verde que transmitam confianca financeira (nao gamificado)
- Configurar CSS variables para dark mode futuro

### Estrutura de pastas minima
```
src/
  app/
    (auth)/        # Grupo de rotas publicas (login/signup) — criar pasta vazia
    (app)/         # Grupo de rotas protegidas — criar pasta vazia

```

### Logging
- Dev: `console.log/warn/error` (suficiente para MVP)

- Sentry: sera adicionado depois, nao e obrigatorio nesta story
