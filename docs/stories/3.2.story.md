## Context

O "momento mágico" do produto. O usuário pergunta em português natural "quanto gastei com delivery esse mês?" e recebe uma resposta com os dados reais. Esta story implementa:

- A interface de chat completa (UI, streaming, histórico)
- A API route `/api/chat` com streaming via Vercel AI SDK
- O context builder que agrega dados financeiros do usuário para o LLM
- Rate limiting server-side (Basic 50/mês Haiku, Pro 200/mês Sonnet)
- **Modo stub:** quando `ANTHROPIC_API_KEY` não configurada, retorna resposta simulada — igual ao padrão estabelecido no Epic 2 para categorização

**A Story 3.1 é pré-requisito** — `usePlan()` deve existir para verificação de plano no frontend.

**Story 3.3** vai adicionar function calling e áudio no Pro por cima desta base.

## User Story

> Como usuário Basic ou Pro,
> quero fazer perguntas sobre minhas finanças em português natural e receber respostas em tempo real,
> para que eu entenda minha situação financeira sem precisar navegar por gráficos.

## Tech Notes for @dev

### Vercel AI SDK — streamText
```typescript
// src/app/api/chat/route.ts
import { streamText } from 'ai'
import { anthropic } from '@ai-sdk/anthropic'

export async function POST(req: Request) {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return new Response('Unauthorized', { status: 401 })

  // Rate limit + plan check
  const { data: profile } = await supabase
    .from('profiles')
    .select('plan, trial_ends_at, audio_enabled, ai_queries_this_month, ai_queries_reset_at')
    .eq('id', user.id)
    .single()

  const tier = getEffectiveTier(profile.plan, profile.trial_ends_at)
  const limit = tier === 'pro' ? 200 : 50
  const model = tier === 'pro' ? 'claude-sonnet-4-6' : 'claude-haiku-4-5-20251001'

  // Reset monthly counter if needed
  const resetAt = new Date(profile.ai_queries_reset_at)
  const startOfThisMonth = new Date(new Date().getFullYear(), new Date().getMonth(), 1)
  if (resetAt < startOfThisMonth) {
    await supabase.from('profiles').update({
      ai_queries_this_month: 0,
      ai_queries_reset_at: startOfThisMonth.toISOString().split('T')[0]
    }).eq('id', user.id)
  } else if (profile.ai_queries_this_month >= limit) {
    return new Response(JSON.stringify({ error: 'rate_limit_exceeded' }), { status: 429 })
  }

  const { messages } = await req.json()
  const context = await buildFinancialContext(user.id, supabase)

  // Stub mode — usar AI SDK Data Stream Protocol (não SSE raw)
  // useChat do Vercel AI SDK espera o formato: 0:"token"\n  d:{...}\n
  if (!process.env.ANTHROPIC_API_KEY) {
    console.log('[chat] stub mode — ANTHROPIC_API_KEY not set')
    const stubText = '[Modo demonstração] Configure ANTHROPIC_API_KEY para ativar o assistente financeiro com seus dados reais.'
    const encoder = new TextEncoder()
    const stream = new ReadableStream({
      start(controller) {
        // AI SDK Data Stream Protocol: text chunk prefix "0:", finish prefix "d:"
        controller.enqueue(encoder.encode(`0:${JSON.stringify(stubText)}\n`))
        controller.enqueue(encoder.encode(`d:${JSON.stringify({ finishReason: 'stop', usage: { promptTokens: 0, completionTokens: 1 } })}\n`))
        controller.close()
      }
    })
    return new Response(stream, {
      headers: {
        'Content-Type': 'text/plain; charset=utf-8',
        'X-Vercel-AI-Data-Stream': 'v1',
      }
    })
  }

  const result = await streamText({
    model: anthropic(model),
    system: buildSystemPrompt(context),
    messages,
    maxTokens: 1024,
  })

  // Increment counter after successful response
  await supabase.from('profiles')
    .update({ ai_queries_this_month: profile.ai_queries_this_month + 1 })
    .eq('id', user.id)

  return result.toDataStreamResponse()
}
```

### Issues

**M1 — Stub mode SSE format incompatível com Vercel AI SDK useChat**
- **Severity:** MAJOR
- **Category:** code
- **Description:** A tech note mostra o stub retornando SSE raw no formato `data: ${JSON.stringify({type:'text-delta', textDelta:...})}\n\n`. O hook `useChat` da Vercel AI SDK espera o **AI SDK Data Stream Protocol**, não SSE genérico. O formato correto é `0:"token"\n` (text chunk) e `d:{...}\n` (done message). O stub atual seria silenciosamente ignorado pelo cliente — o usuário veria loading infinito ou mensagem vazia sem erro.
- **Correction:** O stub deve gerar uma resposta no formato AI SDK Data Stream Protocol. A forma mais simples: usar `createStreamableValue` do AI SDK, ou retornar um `ReadableStream` com o encoding correto: `0:"[Modo demonstração] Configure ANTHROPIC_API_KEY..."\nd:{"finishReason":"stop","usage":{"promptTokens":0,"completionTokens":1}}\n`. Alternativamente: criar um `MockLanguageModelV1` do AI SDK apenas para stub. O @dev deve verificar a versão do AI SDK instalada (`ai` package) para o formato correto.

**M2 — chat_messages table JÁ EXISTE — deferred persistence cria tech debt desnecessária**
- **Severity:** MAJOR
- **Category:** requirements
- **Description:** AC7 diz "se tabela não existir: histórico apenas em memória" e o Out of Scope marca persistência como dívida técnica. Mas `chat_messages` existe em `migration 001_initial_schema.sql` com `role`, `content`, `user_id`, `created_at`. Não usar a tabela que já existe significa: (a) criar tech debt de uma feature já scaffolded; (b) usuário perde histórico ao recarregar, mesmo com infraestrutura pronta. O esforço de persistir é mínimo — um insert/select de poucas linhas.
- **Recommendation:** Rever decisão de scope. Se persistência for mantida fora de escopo por tempo, documentar explicitamente "tabela chat_messages existe e está pronta — implementar persistência é apenas inserts/selects básicos" para que o @dev saiba que o work é trivial quando chegar a vez.

**m1 — Sem estado de erro especificado para falhas de API**
- **Severity:** MINOR
- **Category:** requirements
- **Description:** Não há AC ou menção ao que o usuário vê quando `/api/chat` retorna erro (429 esgotamento, 500, timeout de rede). O `useChat` da Vercel AI SDK expõe `error` no hook — como exibir na UI não está especificado.
- **Recommendation:** Adicionar ao AC1 ou AC2: "Em caso de erro (429, rede), exibir mensagem de erro no chat bubble: '429 → Limite mensal atingido'; outros → 'Erro ao processar sua pergunta. Tente novamente.'"
