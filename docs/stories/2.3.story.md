## Context

Com transacoes categorizadas (Stories 2.1 + 2.2), o dashboard evolui da visualizacao por semana (Story 1.4) para inteligencia financeira por categoria — o "momento de valor" central do plano Free.

Esta e a tela que o usuario abre todo dia. Deve ser bonita, glanceable e impactante. O objetivo e que o usuario entenda sua situacao financeira em 3 segundos (PRD section 3.1).

O componente `summary-cards.tsx` ja existe (Story 1.4) com receitas/despesas/saldo. Esta story adiciona as visualizacoes de categoria sobre os dados ja existentes, sem reescrever o que funciona.

**Decisao de produto:** O dashboard Free e completo e polido. Nao e uma versao degradada — e o produto principal. Comparativo mes-a-mes (FR14) e marcado como Premium no PRD, mas os graficos de categoria atual sao Free.

**PRD refs:** FR10 (dashboard consolidado), FR11 (totais + distribuicao), FR12 (visualmente impactante), FR14 (comparativo — Premium), NFR2 (carrega em < 2s)
**Arch refs:** Component Strategy (section 7.1), Data Fetching (section 7.2)

## Tech Notes for @dev

### Agregacao no servidor (Server Component)
```typescript
// src/app/(app)/dashboard/page.tsx
const supabase = createServerClient()
const { startOfMonth, endOfMonth } = getMonthBounds(selectedPeriod)

// Buscar transacoes do mes (so debitos para categoria)
const { data: expenses } = await supabase
  .from('transactions')
  .select('category, amount')
  .eq('type', 'debit')
  .gte('date', startOfMonth)
  .lte('date', endOfMonth)

// Agregar por categoria em JS (simples para MVP)
const byCategory = expenses?.reduce((acc, t) => {
  const cat = t.category
  acc[cat] = (acc[cat] || 0) + Math.abs(t.amount)
  return acc
}, {} as Record<string, number>) ?? {}

// Passar como prop para componentes de grafico (Client Components)
```

### Query para tendencia (mes anterior)
```typescript
const { startPrev, endPrev } = getPrevMonthBounds(selectedPeriod)
const { data: prevExpenses } = await supabase
  .from('transactions')
  .select('category, amount')
  .eq('type', 'debit')
  .gte('date', startPrev)
  .lte('date', endPrev)
```
> Funções `getMonthBounds` e `getPrevMonthBounds` ja existem em `src/lib/utils/format.ts` (Story 1.4)

### Recharts — Client Components obrigatorio
Recharts usa `useRef` e `ResizeObserver` internamente — DEVE ser Client Component.

```typescript
// src/components/dashboard/category-chart.tsx
'use client'
import { PieChart, Pie, Cell, ResponsiveContainer, Legend } from 'recharts'
import { CATEGORY_CONFIG } from '@/lib/utils/categories'

interface Props {
  data: { category: string; total: number; percentage: number }[]
}

export function CategoryChart({ data }: Props) {
  return (
    <ResponsiveContainer width="100%" height={240}>
      <PieChart>
        <Pie data={data} dataKey="total" cx="50%" cy="50%" innerRadius={60}>
          {data.map((entry) => (
            <Cell key={entry.category} fill={CATEGORY_CONFIG[entry.category]?.color ?? '#9ca3af'} />
          ))}
        </Pie>
        <Legend />
      </PieChart>
    </ResponsiveContainer>
  )
}
```

### Passando dados do Server para Client Component
```typescript
// dashboard/page.tsx (Server) → category-chart.tsx (Client)
// Serializar como plain objects — sem Date, BigInt, etc.
<CategoryChart data={Object.entries(byCategory).map(([category, total]) => ({
  category,
  total,
  percentage: Math.round((total / totalExpenses) * 100)
}))} />
```

- Para MVP: agregacao em JS no servidor e suficiente (500 transacoes = microsegundos)
- Nao criar views materializadas ou funcoes SQL customizadas agora
- Se performance for problema no futuro: otimizar com SQL `GROUP BY` direto
