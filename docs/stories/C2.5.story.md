## Context

Todo brasileiro que declara o Imposto de Renda Pessoa Física (IRPF) precisa informar manualmente despesas dedutíveis. As principais são:
- **Saúde**: médicos, dentistas, hospitais, planos de saúde, exames — deduções ilimitadas
- **Educação**: escolas, faculdades, cursos técnicos — limite anual de R$3.561,50 por dependente (2025)

O problema: o usuário precisa juntar notas e comprovantes manualmente, mês a mês, ao longo do ano. Nenhum aplicativo de finanças pessoais brasileiro resolve isso diretamente — é uma "white space" identificada no market research.

A solução é simples: as categorias `saude` e `educacao` já existem no Finansim. As transações já estão categorizadas pela IA. Basta mapear essas categorias para os campos da declaração do IR, filtrar por ano-calendário, e exportar um relatório organizado.

**Campos do IR mapeados:**

| Categoria Finansim | Ficha do IRPF | Dedução |
|-------------------|--------------|---------|
| `saude` | Pagamentos / Despesas Médicas e de Saúde | Ilimitada |
| `educacao` | Pagamentos / Instrução | Limitada (R$3.561,50/dependente/ano) |

**Escopo deliberadamente simples:** sem integração com Receita Federal, sem pré-preenchimento de DIRF, sem validação de CNPJ/CPF do prestador. Apenas: "aqui estão suas transações de saúde e educação, organizadas para você".

---

## User Story

> Como usuário que declara IRPF,
> quero ver um relatório das minhas despesas de saúde e educação organizadas por campos da declaração do Imposto de Renda,
> para preencher minha declaração rapidamente sem precisar vasculhar comprovantes ou extratos manualmente.

## Tech Notes for @dev

### Mapeamento de categorias para IR

```typescript
// src/lib/irpf/category-mapping.ts

export const IRPF_CATEGORIES = ['saude', 'educacao'] as const;
export type IRPFCategory = typeof IRPF_CATEGORIES[number];

export const IRPF_FIELD_LABELS: Record<IRPFCategory, string> = {
  saude: 'Despesas Médicas e de Saúde',
  educacao: 'Instrução',
};

export const IRPF_LIMITS: Record<IRPFCategory, number | null> = {
  saude: null,           // ilimitado
  educacao: 3561.50,     // limite 2025 — atualizar anualmente
};

export const IRPF_FICHA: Record<IRPFCategory, string> = {
  saude: 'Pagamentos > Despesas Médicas e de Saúde',
  educacao: 'Pagamentos > Instrução',
};
```

### Geração de CSV com BOM UTF-8

```typescript
// src/lib/irpf/export-csv.ts
export function generateIRPFCsv(data: IRPFData): string {
  const BOM = '\uFEFF';
  const header = 'Data,Descrição,Categoria,Ficha IRPF,Valor (R$)\n';
  
  const rows = [
    ...data.saude.transactions.map(t => formatRow(t, 'saude')),
    ...data.educacao.transactions.map(t => formatRow(t, 'educacao')),
  ].join('\n');
  
  return BOM + header + rows;
}

function formatRow(t: Transaction, category: IRPFCategory): string {
  const date = new Date(t.date).toLocaleDateString('pt-BR');
  const description = `"${t.description.replace(/"/g, '""')}"`;
  const categoryLabel = IRPF_FIELD_LABELS[category];
  const ficha = IRPF_FICHA[category];
  const value = t.amount.toFixed(2).replace('.', ',');
  return `${date},${description},${categoryLabel},${ficha},${value}`;
}
```

### Filtro de ano nas queries

```typescript
// Para buscar transações do ano especificado
const { data } = await supabase
  .from('transactions')
  .select('date, description, amount')
  .eq('user_id', userId)
  .eq('type', 'debit')
  .in('category', ['saude', 'educacao'])
  .gte('date', `${year}-01-01`)
  .lte('date', `${year}-12-31`)
  .order('date', { ascending: true });
```

### Anos disponíveis

Para popular o seletor de ano, buscar o `MIN(date)` das transações do usuário e gerar o range de anos desde então até o ano atual.

## Out of Scope

- Integração com e-CAC ou Receita Federal
- Pré-preenchimento automático da DIRF ou declaração
- Validação de CNPJ do prestador de saúde
- Subcategorias de saúde (médico vs. farmácia vs. plano de saúde) — a categoria `saude` é tratada como um bloco único
- Dependentes (o IR de dependentes requer dados extras fora do escopo)
- Categorias adicionais além de `saude` e `educacao` (ex: `doacoes` — não existe como categoria no Finansim)
- Atualização automática do limite de educação quando a Receita Federal alterar (campo hardcoded — manter anotação para atualizar anualmente)
- Histórico de anos anteriores a 2025 (funciona para qualquer ano com dados, mas UI e limite são otimizados para 2025/2026)

#### MINOR

**mn1 — Limite de educação hardcoded para 2025 em contexto de 2026**
O valor `R$3.561,50` é o limite do IRPF 2025 (declaração entregue em 2026). A story foi escrita em 2026 e usuários futuramente vão querer declarar o ano-calendário 2026 (limite ainda não definido pela Receita Federal). O @dev deve estruturar o limite como constante nomeada por ano:
```typescript
const EDUCATION_LIMITS: Record<number, number> = {
  2025: 3561.50,
  // 2026: atualizar quando Receita Federal divulgar
};
```
E selecionar conforme o `year` do request. Adicionar comentário: `// Verificar e atualizar anualmente conforme tabela IRPF`.
